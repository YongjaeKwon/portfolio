# 포트폴리오 업무 총정리

> 이 문서는 포트폴리오 본문 반영을 위한 원본 정리입니다. 공개 포트폴리오에는 내부 저장소 URL, 운영 서버, 고객사별 상세 정책, 계정, 설정값, 리비전 번호를 직접 노출하지 않는 것을 권장합니다.

## 최종 포지셔닝

운영 중인 B2B/B2G 시스템에서 화면, API, SQL, 외부 연동을 함께 다루며 사용자 업무 흐름을 완성해 온 프론트엔드 중심 웹 개발자입니다.

단순히 화면만 구현한 것이 아니라, 사용자가 실제로 거치는 접수, 동의, 첨부, 전자서명, 엑셀 업로드, 메일 발송, 제출 현황 조회, 권한별 조회 조건, 통계/대시보드 확인까지 이어지는 업무 단계를 화면과 데이터 흐름으로 연결해 왔습니다.

## 내가 실제로 해온 일의 유형

### 1. 운영자가 반복해서 쓰는 관리자 화면 개발

- 교육관리, CE 현황, 협력사 등록, 자산현황, 서비스 리포트, KPI, 공지/설문 관리자 화면
- 목록, 상세, 팝업, 엑셀, 메일, 검색 조건, 제출 현황이 이어지는 구조 구현
- Tabulator, Nexacro Grid, WebSquare 화면에서 행 선택, 필터, 페이징, 다운로드 흐름 처리

면접 표현:

> 운영자가 반복적으로 사용하는 관리자 화면을 많이 다뤘습니다. 단순 목록 화면보다 등록, 업로드, 발송, 제출 현황, 엑셀 다운로드까지 이어지는 업무 흐름을 주로 구현했습니다.

### 2. API와 SQL까지 같이 보는 프론트엔드 구현

- 화면 요구사항에 맞춰 Controller, Service, MyBatis Mapper를 함께 수정
- 권한, 기관, 부서, CE, 상태, 기간, 사업 차수 같은 조회 조건 처리
- 화면 필터와 SQL 조회 결과가 어긋나는 문제 재현 및 수정

면접 표현:

> 화면에서 보이는 조건이 실제 SQL 조회 조건과 맞는지 함께 확인하면서 개발했습니다. 그래서 프론트엔드 구현 과정에서도 API 응답 구조와 데이터 조건을 같이 보는 편입니다.

### 3. 외부 연동 결과를 화면 상태로 연결

- 메일 발송, 알림톡 발송, OTP 인증, QR 확인, 엑셀 업로드, 파일 등록, 전자서명
- 성공/실패/예외/재시도/잠금 상태를 화면에 반영
- 발송 완료 후 선택 잠금, 업로드 검증 결과 표시, OTP 예외 분기 등 처리

면접 표현:

> 외부 연동 기능은 요청 성공만으로 끝나지 않기 때문에, 발송 결과나 인증 상태가 실제 화면에서 어떻게 보여야 하는지까지 함께 처리했습니다.

### 4. 공통화와 재사용

- 댓글/대댓글 공통 Vue 컴포넌트
- 센터 검색 공통 컴포넌트
- 파일 등록 모달 통일
- 그리드 옵션/체크박스/라디오 등 공통 UI 개선

면접 표현:

> 비슷한 UI와 로직이 여러 화면에 반복될 때는 공통 컴포넌트나 공통 스크립트로 분리해 재사용할 수 있게 정리했습니다.

### 5. 운영 이슈 대응

- 검색 조건 누락
- 엑셀 다운로드 조건 불일치
- 특정 권한/기관/센터에서만 발생하는 조회 오류
- 로그인/OTP 예외
- 날짜 마감 조건
- 화면 표시 오류
- 데이터 중복 조회

면접 표현:

> 운영 이슈는 단일 파일만 보고 끝내기보다, 화면 조건, API 파라미터, SQL 조건, 실제 사용 권한을 함께 확인하며 원인을 좁혔습니다.

## 프로젝트별 아키텍처 비교

### PPS

```text
HTML/Thymeleaf + Vue.js
  -> Spring Boot Controller
  -> Service
  -> MyBatis Mapper
  -> DB1/DB2
  -> Email / Excel / OTP / File
```

강점:

- 현대적인 Spring Boot 기반 운영 포탈 경험
- Vue 화면 로직과 백엔드/API/SQL을 함께 다룬 경험
- React/TypeScript 지원 시에도 관리자 화면 상태 관리 경험으로 연결 가능

### TSMS / IDCMS / AS 화면

```text
WebSquare / JSP / Mobile / Tablet
  -> Java Controller
  -> Service / DAO
  -> MyBatis Mapper
  -> 업무 DB
  -> QR / 알림톡 / 전자서명 / 공개 페이지
```

강점:

- 모바일/태블릿/데스크톱이 연결되는 업무 흐름 경험
- QR, 개인정보 동의, 알림톡, 전자서명처럼 상태 전이가 중요한 기능 경험

### 교육청 IT 자산관리

```text
JSP / Tabulator Dashboard
  -> Spring MVC Controller
  -> Service / DAO
  -> MyBatis Mapper
  -> 자산/기관/상태/이력 데이터
  -> 대시보드 집계
```

강점:

- 대시보드와 현황 화면 경험
- 권한/기관/사업 차수/자산 상태에 따라 달라지는 조회 조건 처리 경험

### SR30

```text
Nexacro
  -> Spring MVC
  -> Service / Mapper
  -> MyBatis / Oracle
  -> 리포트 / KPI / 엑셀 / 이력 조회
```

강점:

- 운영 개선과 통계/리포트 화면 경험
- 오래된 업무 시스템에서 화면과 SQL을 함께 개선한 경험

## 이력서/포트폴리오용 통합 소개문

운영 중인 B2B/B2G 업무 시스템에서 Vue.js, WebSquare, JSP, Nexacro 기반 화면과 Java/Spring API, MyBatis SQL을 함께 다뤄왔습니다. 협력사 포탈에서는 교육관리, 엑셀 업로드, 메일 발송, 댓글 공통화, OTP 예외 처리를 개발했고, AS/자산관리/물류 시스템에서는 모바일 접수, 개인정보 동의, QR 확인, 알림톡 발송, 대시보드, 통계/리포트 화면을 개선했습니다. 화면 구현에 그치지 않고 API 응답, 권한 조건, 조회 SQL, 외부 연동 결과가 실제 사용자 흐름과 맞는지 함께 확인하며 개발합니다.

## 강점 문장 후보

- 운영자가 반복해서 사용하는 관리자 화면을 등록, 업로드, 발송, 조회, 엑셀 다운로드까지 이어지는 흐름으로 구현했습니다.
- 화면 요구사항만 보는 것이 아니라 API 응답, 권한 조건, MyBatis SQL을 함께 확인하며 화면과 데이터 정합성을 맞췄습니다.
- 메일, 알림톡, OTP, QR, 파일, 엑셀처럼 외부 처리 결과가 섞이는 기능에서 성공/실패/예외 상태를 화면에 반영했습니다.
- 게시판/설문 댓글, 센터 검색, 파일 등록 모달처럼 반복되는 UI와 처리 로직을 공통화했습니다.
- 운영 이슈는 실제 사용자 권한, 검색 조건, 화면 상태, SQL 조회 결과를 함께 재현하며 원인을 찾았습니다.

## 면접에서 설명할 때의 핵심 스토리

1. 화면만 만든 것이 아니라 업무 흐름을 연결했다.
2. 프론트엔드 구현 과정에서 API와 SQL까지 같이 봤다.
3. 운영 시스템이라 예외 케이스와 권한 조건이 중요했다.
4. 반복되는 화면/로직은 공통화하려고 했다.
5. 정량 성과가 부족한 프로젝트는 커밋 수보다 문제의 복잡도와 검증 방식을 설명한다.
