# SR30 물류/서비스 운영 시스템 업무 정리

> 공개 포트폴리오 반영 전 정리 문서입니다. 내부 운영 URL, 고객사별 상세 정책, 계정 및 서버 정보는 제외했습니다.

## 한 줄 요약

Nexacro와 Spring MVC 기반 물류/서비스 운영 시스템에서 일정 공유, 설문 페이지, 물류/재고 조회, 서비스 리포트, KPI, 우정사업 현황판, 접속/개인정보 이력, 엑셀 다운로드 등 운영 개선 작업을 폭넓게 수행했습니다.

## 내가 맡은 역할

- Nexacro 화면 수정 및 신규 화면 구성
- Spring MVC Service/Mapper와 MyBatis SQL 수정
- 물류, 서비스, 통계, 리포트, 관리자 메뉴의 운영 이슈 대응
- 엑셀 다운로드, 조회 조건, 집계 SQL, KPI 지표 등 운영 데이터 화면 개선
- 설문/캠페인성 화면과 관리자 확인 화면 구성

## 주요 업무 내용

### 1. 일정 공유 및 작업 스케줄

운영자가 작업 일정을 공유하고 진행 상태를 확인할 수 있는 일정관리 화면을 개발했습니다.

- 일정 공유 화면 신규 구성
- 일정 공유 1차 완성본 개발
- 작업 스케줄 캘린더 제목 표기 변경
- 진행 상태와 제목이 함께 보이도록 표시 방식 변경
- 제목/내용 입력 길이 제한 반영
- 실제 마감 일자 DB 입력 수정
- 일정관리 명칭을 작업 스케줄 기준으로 정리

포트폴리오 표현 후보:

> 운영자가 작업 일정을 공유하고 진행 상태를 확인할 수 있도록 Nexacro 일정관리 화면과 백엔드 조회/저장 흐름을 구성했습니다.

### 2. 설문 및 외부 응답 화면

설문 페이지와 응답 화면을 구성하고, 운영자가 응답 현황을 확인할 수 있도록 화면과 정적 리소스를 정리했습니다.

- 설문 폴더 및 화면 리소스 생성
- 설문 페이지 완성
- 추천 답변 및 화면 문구 변경
- XSS 치환 처리
- 설문 참여 전/진행/완료 상태를 구분하는 화면 구성

포트폴리오 표현 후보:

> 외부 사용자가 접근하는 설문 페이지를 구성하고, 응답 상태와 완료 화면이 자연스럽게 이어지도록 화면 리소스와 처리 흐름을 정리했습니다.

### 3. 물류/재고/수불 조회 개선

물류 자재와 재고 화면에서 조회 조건, SQL, 엑셀 다운로드, 지표 표기를 개선했습니다.

- 재고 검증 화면 재배치
- 수불 데이터 SQL 수정
- SQL index 형변환 제거
- 종합 수불 내역에 RMA 건 포함
- 창고재고조회 엑셀 다운로드 수정
- 회계 수불부 엑셀 다운로드 수정
- 공급율/처리율 지수, 선배치 현황, 불량회수율 지표 표기 변경
- 메이커별 소계 표기

포트폴리오 표현 후보:

> 물류/재고 조회 화면에서 SQL 조건, 수불 집계, 엑셀 다운로드, 지표 표기를 개선해 운영자가 재고와 수불 데이터를 더 정확하게 확인할 수 있도록 수정했습니다.

### 4. 서비스 종합현황 및 리포트

서비스 운영 화면에서 접수, 수리, 리포트, KPI 관련 조회 조건과 출력 화면을 개선했습니다.

- 서비스 일괄 접수 컬럼명 변경
- AS 수리의뢰서 및 접수증 화면/SQL 수정
- 서비스 종합현황 전화번호 검색 오류 수정
- Serial No 매핑 수정
- 접수 대기 기간 범위 설정
- 서비스 내역서 센터명 표기 수정
- 원불량 처리 접수번호 활성화
- 레노버/후지필름/Epson 등 메이커별 화면 조건 수정

포트폴리오 표현 후보:

> 서비스 종합현황과 리포트 화면에서 검색 조건, 접수번호, 센터명, 메이커별 분기 조건을 수정해 운영자가 접수/수리 데이터를 정확히 조회할 수 있도록 개선했습니다.

### 5. KPI 및 통계 지표

서비스 리포트와 KPI 화면에 새로운 지표와 조회 조건을 추가했습니다.

- Epson 서비스 Report 지표 추가
- 48H 달성율 지표 추가
- KPI 대상 문구 수정
- 담당자별 추천 지표 추가
- 앱손 기간별 서비스 건수 KPI 신규
- 후지필름 KPI 화면 및 SQL 구성
- Epson Call Report 데이터셋 바인드 누락 수정
- Raw Data 컬럼 추가

포트폴리오 표현 후보:

> 서비스 리포트와 KPI 화면에서 48H 달성율, 기간별 서비스 건수, 담당자별 지표 등 운영 통계 항목을 추가하고 SQL/화면 바인딩을 함께 수정했습니다.

### 6. 우정사업 현황판 및 보고서

현황판, 보고서, 설치/철거 등록 화면을 구성하고 검색 조건과 집계 SQL을 개선했습니다.

- 현황판 화면 구성
- 보고서 관리 화면 구성
- 기간별 완료 현황 수정
- 설치/철거 등록 SQL 및 화면 추가
- 기간별 철거 현황 검색 조건 추가
- 재사용 여부, 재배치 항목 추가
- 보고서 검색 조건과 원장합계 조건 개선

포트폴리오 표현 후보:

> 현황판과 보고서 관리 화면을 구성하고 설치/철거, 완료 현황, 재사용 여부 같은 운영 조건을 SQL과 Nexacro 화면에 반영했습니다.

### 7. 관리자 이력 및 개인정보 조회

운영 관리자가 접속 이력, 개인정보 다운로드 이력, 데이터 삭제 건을 확인할 수 있도록 관리 화면을 개선했습니다.

- 접속 내역 조회 화면 추가
- 개인정보 다운로드 내역 조회 화면 추가
- 화면별 접속 횟수 내역 조회
- 화면 접속기록 검색 조건 수정
- 데이터 삭제 처리 건 조회
- 잔여 데이터 건수 추가
- 데이터 삭제건, 사용자별 접속 건, 화면별 접속 횟수, 개인정보 다운로드 횟수 엑셀 다운로드 추가

포트폴리오 표현 후보:

> 접속 이력, 화면별 접속 횟수, 개인정보 다운로드 이력, 데이터 삭제 건 조회 화면을 개선해 운영 감사와 이력 확인에 필요한 데이터를 조회할 수 있도록 했습니다.

## 기술적 난이도

- Nexacro 화면 파일, Java Service/Mapper, MyBatis SQL을 함께 수정해야 하는 작업이 많았습니다.
- 운영 화면은 고객사/메이커/센터/기간/상태 조건이 복잡해 검색 조건 하나가 통계 결과에 영향을 줄 수 있었습니다.
- 엑셀 다운로드와 통계 화면은 화면에 보이는 조건과 SQL 집계 기준이 반드시 일치해야 했습니다.
- 운영 중인 시스템이라 신규 지표 추가와 기존 화면 오류 수정이 동시에 진행되었습니다.

## 아키텍처 요약

```text
운영자 Nexacro 화면
  -> Spring MVC Controller/Service
  -> Mapper Interface
  -> MyBatis SQL
  -> Oracle 기반 운영 데이터
  -> 통계/리포트/엑셀 다운로드/이력 조회
```

주요 구성:

- Frontend: Nexacro, XFDL 화면
- Backend: Java, Spring MVC
- Database: Oracle, MyBatis, PL/SQL 성격의 통계/집계 SQL
- Operation: 엑셀 다운로드, 접속 이력, 개인정보 다운로드 이력, KPI/리포트
