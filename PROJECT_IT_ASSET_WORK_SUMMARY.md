# 교육청 IT 자산관리 솔루션 업무 정리

> 공개 포트폴리오 반영 전 정리 문서입니다. 내부 저장소 URL, 기관별 상세 정책, 운영 서버 정보는 제외했습니다.

## 한 줄 요약

교육청과 학교의 IT 자산을 관리하는 Spring MVC 기반 업무 시스템에서 자산 대시보드, 자산현황, 유상처리 현황, 검색 조건, 집계 배치, 권한/기관별 조회 화면을 개발하고 개선했습니다.

## 내가 맡은 역할

- 자산관리 대시보드와 자산현황 화면 개발 및 개선
- 기관/사업 차수/자산 상태별 조회 조건과 집계 화면 구현
- JSP 화면, Java Controller/Service/DAO, MyBatis SQL을 함께 수정
- 대시보드 카드 클릭 시 상세 화면으로 이동하는 화면 흐름 구성
- 검색 조건 누락, 날짜 형식, 숫자 자리수, 그리드 컬럼 등 운영 이슈 수정

## 주요 업무 내용

### 1. 자산관리 대시보드

사업 차수, 자산 상태, 분실/폐기 현황 등 운영자가 빠르게 확인해야 하는 정보를 대시보드로 정리했습니다.

- 자산관리 대시보드 클릭 시 보이는 화면 분기
- 사업 차수별 자산 현황 집계
- 사업 차수별 분실/폐기 현황 집계
- 지역별/기관별 대시보드 화면 추가 및 수정
- 대시보드 차트 색상과 표시 방식 변경
- 대시보드 카드 클릭 시 관련 현황 화면으로 이동하는 흐름 구현
- 숫자 자리수 표기와 날짜 형식 변경

포트폴리오 표현 후보:

> 자산관리 대시보드에서 사업 차수, 자산 상태, 분실/폐기 현황을 집계하고, 카드 클릭 시 관련 현황 화면으로 이동하도록 화면/API/SQL 흐름을 구성했습니다.

### 2. 자산현황 화면

자산 목록과 현황 화면에서 검색 조건, 그리드 컬럼, 조회 SQL을 개선했습니다.

- 자산 등록 조회 쿼리 변경
- 자산현황표 데이터 변경
- 메인 그리드의 불필요 컬럼 제외
- 자산현황 검색 조건 누락 수정
- 조회 조건에 따라 화면과 SQL 결과가 일치하도록 보정

포트폴리오 표현 후보:

> 자산현황 화면에서 검색 조건 누락과 그리드 컬럼 문제를 수정하고, 화면에 보이는 조건과 MyBatis SQL 조회 조건이 같은 기준으로 동작하도록 개선했습니다.

### 3. 유상처리 현황 신규 생성

자산 또는 AS 업무에서 유상처리 상태를 별도로 조회할 수 있는 현황 화면을 신규로 구성했습니다.

- 유상처리 현황 화면 신규 생성
- Controller, Service, DAO, Mapper 흐름 추가
- 그리드 입력 길이 제한 추가
- 바로가기 아이콘 링크 추가
- 관련 화면과 조회 SQL 연결

포트폴리오 표현 후보:

> 유상처리 현황 메뉴를 신규로 구성하면서 JSP 화면, Controller, Service, DAO, MyBatis Mapper를 연결해 운영자가 별도 현황으로 확인할 수 있도록 개발했습니다.

### 4. 집계 배치 및 이력 화면

대시보드 집계와 이력 확인을 위한 배치/이력 흐름을 개선했습니다.

- 대시보드 집계 배치 수정
- 자산 메뉴 리뉴얼 과정에서 현황 메뉴 별도 생성
- 이력 history 구조 추가
- 자산 현황과 대시보드 집계가 이어지도록 SQL과 화면 흐름 정리

포트폴리오 표현 후보:

> 자산 대시보드 집계 배치와 이력 화면을 함께 정리해 운영자가 현재 현황뿐 아니라 변경 이력까지 확인할 수 있는 구조로 개선했습니다.

### 5. 조직/권한 기반 조회 이슈 대응

조직 트리, 기관별 화면, 권한별 조회 조건에서 발생하는 운영 이슈를 수정했습니다.

- 조직관리 row select 시 하위 노드가 사라지는 현상 수정
- 기관별 대시보드 화면 구성
- 기관/사업 차수/자산 상태에 따라 조회 결과가 달라지는 조건 처리
- nav 클릭 시 자산 정보가 올바르게 변경되도록 수정

포트폴리오 표현 후보:

> 조직/기관 기준으로 조회 범위가 달라지는 화면에서 트리 선택, 메뉴 이동, 검색 조건이 서로 어긋나지 않도록 운영 이슈를 수정했습니다.

## 기술적 난이도

- 단순 화면 표시가 아니라 JSP, Controller, Service, DAO, MyBatis Mapper를 한 번에 맞춰야 하는 작업이 많았습니다.
- 대시보드 집계와 목록 조회가 같은 기준을 사용해야 해 SQL 조건과 화면 필터 정합성이 중요했습니다.
- 기관/사업/자산 상태에 따라 조회 범위가 달라지는 구조라 검색 조건 누락이 실제 운영 수치 오류로 이어질 수 있었습니다.
- 운영 중인 시스템에서 신규 메뉴를 추가하면서 기존 메뉴와 바로가기, nav, 대시보드 이동 흐름도 함께 조정해야 했습니다.

## 아키텍처 요약

```text
관리자 브라우저
  -> JSP / Tabulator 기반 업무 화면
  -> Spring MVC Controller
  -> Service / DAO
  -> MyBatis Mapper
  -> 자산/기관/상태/이력 데이터
  -> 대시보드 집계 및 현황 조회
```

주요 구성:

- Frontend: JSP, JavaScript, Tabulator, 대시보드 화면
- Backend: Java, Spring MVC
- Database: MyBatis, MariaDB 계열 조회/집계 SQL
- Batch/Operation: 대시보드 집계 배치, 변경 이력 조회
