# PPS 협력사 포탈 시스템 업무 정리

> 공개 포트폴리오 반영 전 정리 문서입니다. 내부 저장소 URL, 운영 서버, 계정, 고객사별 상세 정책, 설정값은 의도적으로 제외했습니다.

## 한 줄 요약

협력사, CE, 교육, 공지, 설문, 유니폼/소모품 신청 데이터를 다루는 Spring Boot 기반 운영 포탈에서 관리자 화면, Vue 기반 화면 로직, MyBatis 조회 조건, 메일 발송, 엑셀 업로드/다운로드, OTP 인증 예외 처리를 개발했습니다.

## 내가 맡은 역할

- 운영자가 반복적으로 사용하는 관리자 화면의 기능 개발과 운영 이슈 대응
- Thymeleaf/HTML 템플릿 안에서 동작하는 Vue.js 화면 로직 구현
- Spring Boot Controller/Service와 MyBatis SQL을 함께 수정해 화면, API, 데이터 조회 흐름 정합성 확보
- 엑셀 업로드/다운로드, 메일 발송, 파일 처리, 인증 예외처럼 운영 중 실패 케이스가 생기기 쉬운 기능의 상태 처리
- 게시판, 설문, 공지, 제안하기 등 여러 화면에서 반복되는 UI/처리 흐름 공통화

## 주요 업무 내용

### 1. 교육관리 기능

교육 등록, 대상자 관리, 사용자 제출 화면, 관리자 현황 화면, 엑셀 업로드/다운로드, 교육 메일 발송까지 이어지는 교육 운영 흐름을 구현했습니다.

- 교육 대상자 조회/저장 API 흐름 구현
- 교육 대상자 관리 화면과 사용자 제출 화면 개발
- 센터/부서/CE 기준 대상자 조회 조건 처리
- 교육 대상자 엑셀 양식 다운로드와 엑셀 업로드 로직 변경
- 업로드 데이터 검증 후 대상자 목록과 제출 여부가 이어지도록 화면 상태 처리
- 교육 등록 시 대상자에게 안내 메일을 발송하는 흐름 추가
- 교육 상세 현황 엑셀 다운로드 기능 추가

포트폴리오 표현 후보:

> 교육 등록부터 대상자 엑셀 업로드, 대상자 저장, 안내 메일 발송, 제출 현황 조회까지 이어지는 관리자 업무 흐름을 Vue 화면, Spring Boot API, MyBatis SQL로 구현했습니다.

### 2. 협력사/파트너 관리

협력사 등록과 조회 화면에서 검색 조건, 입력 형식, 담당자 정보, 정산 유형, 계정 연동 관련 화면 상태를 개선했습니다.

- 파트너 등록 화면 검색 조건 및 조회 SQL 개선
- 다중/단일 셀렉트 UI 구분 적용
- 휴대폰 입력 형식 변경
- 담당자 정보 그리드 및 삭제 토글 흐름 개선
- 담보 금액 필수값 처리
- 정산 유형에 따라 알림톡 발송 대상에서 제외되는 조건 처리
- 센터 검색 공통 컴포넌트 생성 및 여러 화면에 적용

포트폴리오 표현 후보:

> 협력사 등록 화면에서 검색 조건, 담당자 정보, 입력 검증, 센터 검색 공통 컴포넌트를 정리해 운영자가 반복적으로 처리하는 등록/수정 흐름의 오류 가능성을 줄였습니다.

### 3. CE 및 협력사 현황 관리

CE 관리와 현황 화면에서 담당 업무, 검증 상태, 엑셀 다운로드 조건, 메일 발송 흐름을 개선했습니다.

- CE 담당 업무 필수값 및 표시 방식 개선
- CE 등록 시 생년월일 형식과 사용 여부 처리 개선
- 검증/미검증 상태별 조회 조건 추가
- CE 현황 엑셀 다운로드 조회 조건 보완
- CE 현황 메일 발송 기능 추가
- 현황 메일 발송 이력 저장을 위한 메일 로그 흐름 추가

포트폴리오 표현 후보:

> CE 현황 화면에서 검증 상태, 담당 업무, 엑셀 다운로드 조건, 메일 발송 흐름을 정리해 운영자가 대상자 상태를 더 명확히 확인하고 후속 안내를 진행할 수 있도록 개선했습니다.

### 4. 게시판, 설문, 댓글 공통화

게시판과 설문 화면에 반복되던 댓글/대댓글 처리 흐름을 공통 컴포넌트로 분리하고, 여러 화면에서 같은 방식으로 작성/수정/삭제가 가능하도록 정리했습니다.

- 게시판 댓글 및 대댓글 작성, 조회, 삭제 기능 구현
- 댓글 공통 Vue 컴포넌트 생성
- 게시판, 게시판 관리자, 설문, 설문 관리자 화면에 공통 댓글 흐름 적용
- 설문 응답 버튼, 설문 마감 후 수정 가능 여부, 입력 자동완성, 삭제 버튼 상태 등 화면 이슈 개선
- 파일 등록 모달 UI 흐름을 여러 게시성 화면에서 통일

포트폴리오 표현 후보:

> 게시판과 설문 화면에 흩어져 있던 댓글/대댓글 처리 로직을 공통 Vue 컴포넌트로 정리해 작성, 수정, 삭제, 답글 조회 흐름을 여러 화면에서 재사용할 수 있게 했습니다.

### 5. 공지사항, 정책공지, 제안하기

공지와 제안하기 화면에서 읽음 여부, 파트너 검색, 메일 발송, 목록 조회 조건을 개선했습니다.

- 공지사항 읽음 여부 저장
- 정책공지 읽은 대상 목록 조회 기능 추가
- 공지/정책공지 파트너 검색 조건 수정
- 제안하기 등록/수정 검증 및 메일 발송 흐름 개선
- 메인 화면에 노출되는 제안하기 조회 조건 변경

포트폴리오 표현 후보:

> 공지사항과 정책공지에서 읽음 여부를 저장하고 읽은 대상 목록을 조회할 수 있도록 화면/API/SQL 흐름을 추가했습니다.

### 6. 유니폼/소모품 신청

유니폼/소모품 신청 화면에서 입력 검증, 담당자 메일 발송, 메일 템플릿, 환경별 수신자 설정을 처리했습니다.

- 유니폼 신청 시 담당자 메일 발송 기능 추가
- 유니폼 메일 템플릿 생성
- 신청 화면 입력 필수값 보완
- 영문명 입력 필수 처리
- 환경별 담당자 이메일 설정 변경

포트폴리오 표현 후보:

> 유니폼/소모품 신청 화면에서 신청 데이터 검증과 담당자 메일 발송을 연결하고, 환경별 수신자 설정과 메일 템플릿을 함께 정리했습니다.

### 7. 로그인, OTP, 인증 예외 처리

본사/협력사 계정 로그인 흐름에서 OTP 예외 조건, IP 기준 분기, 실패 문구, 비밀번호 초기화 모달을 개선했습니다.

- Google OTP IP 예외 처리
- 본사 로그인 시 IP 대조 후 OTP 우회/정규화 흐름 개선
- 미사용 계정 로그인 실패 문구 변경
- 메인 로그인 화면 비밀번호 초기화 모달 수정
- 개발 서버 OTP 예외 및 테스트 로그 정리

포트폴리오 표현 후보:

> Google OTP 인증 흐름에서 IP 기반 예외 조건과 본사 계정 분기 처리를 추가하고, 인증 상태에 따라 로그인 후 화면 전환이 어긋나지 않도록 개선했습니다.

## 기술적 난이도

- 화면만 수정한 것이 아니라 Controller, Service, MyBatis Mapper, HTML/Vue 템플릿을 함께 수정해야 하는 작업이 많았습니다.
- 엑셀 업로드, 메일 발송, OTP 인증, 댓글 공통화처럼 성공/실패/예외 상태를 분리해야 하는 기능이 많았습니다.
- 같은 데이터가 관리자 화면, 사용자 화면, 엑셀, 메일, 제출 현황에서 일관되게 보여야 했기 때문에 조회 조건과 화면 상태 정합성이 중요했습니다.
- 운영 중인 시스템이라 기능 추가와 함께 기존 화면의 영향 범위를 계속 확인해야 했습니다.

## 아키텍처 요약

```text
사용자/관리자 브라우저
  -> Thymeleaf/HTML 템플릿 + Vue.js 화면 로직
  -> Spring Boot Controller
  -> Service 계층
  -> MyBatis Mapper
  -> DB1/DB2 데이터 조회 및 저장
  -> 메일/알림/파일/엑셀 처리 유틸
```

주요 구성:

- Frontend: Vue.js, HTML template, Tabulator, 공통 Vue 컴포넌트
- Backend: Java, Spring Boot, Spring Security
- Database: MyBatis Mapper, 운영 DB 조회 조건
- External/Utility: Email, Excel upload/download, OTP, file handling

## 포트폴리오에 넣을 최종 요약 문장

PPS 협력사 포탈에서는 교육관리, 협력사 등록, CE 현황, 게시판/설문, 공지, 유니폼 신청, OTP 인증 흐름을 개발했습니다. 특히 교육관리에서는 대상자 엑셀 업로드, 대상자 저장, 안내 메일 발송, 제출 현황 조회까지 이어지는 운영 흐름을 Vue 화면, Spring Boot API, MyBatis SQL로 구현했고, 게시판/설문에서는 댓글·대댓글 처리 로직을 공통 컴포넌트로 정리해 여러 화면에서 재사용할 수 있도록 개선했습니다.
