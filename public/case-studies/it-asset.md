# 교육청 IT 자산관리 솔루션

> 공개용 정리입니다. 내부 코드, 고객사별 상세 정책, 운영 URL, 환경 정보는 제외했습니다.

## 개요

교육청과 학교의 IT 자산을 등록, 배포, 대여, 반납, 폐기, P2P 이동 단계로 관리하는 Spring MVC 기반 업무 시스템입니다. 교육청, 학교, 부서 권한에 따라 조회 범위가 달라지는 점이 핵심입니다.

## 담당 역할

- 자산 등록·대여·반납·폐기 및 P2P 이동/승인 화면 개선
- 교육청 > 학교 > 부서 권한 계층에 따른 조회 범위와 검색 조건 처리
- 대시보드·통계 화면의 상태별 집계와 동적 컬럼 조회 개선
- SMS, 메일, 우편번호, 외부 AS 연동 결과를 화면 흐름에 반영
- MyBatis SQL 조회 조건과 화면 검색 조건 정합성 점검

## 담당 메뉴 범위

- 자산 등록/수정: 기본 정보, 위치, 보유 기관, 상태값 입력과 변경
- 대여/반납/폐기: 자산 상태 변경, 이력 조회, 처리 결과 확인
- P2P 이동/승인: 학교 또는 부서 간 이동 요청, 승인 상태 조회
- 대시보드/통계: 상태별 자산 수량, 기관별 집계, 동적 컬럼 기반 조회
- 외부 연동: 우편번호 조회, SMS/메일 발송, 외부 AS 연동 결과 표시

## 권한·조회 조건 예시

- 교육청 권한은 관할 학교와 부서의 자산을 함께 조회할 수 있도록 상위 기관 기준 조건을 적용했습니다.
- 학교 권한은 소속 학교의 자산과 부서별 세부 조건을 기준으로 목록을 제한했습니다.
- 부서 권한은 담당 부서 또는 사용자에게 연결된 자산만 보이도록 화면 검색 조건과 SQL 조건을 함께 점검했습니다.

## 상태·예외 처리

- 자산 상태가 등록, 대여, 반납, 폐기, 이동 요청, 승인 완료로 바뀔 때 화면 액션과 조회 결과가 함께 바뀌도록 처리했습니다.
- 외부 API 연동 결과는 성공, 실패, 재시도 필요 상태를 화면에서 확인할 수 있도록 반영했습니다.
- 권한 조건 변경 후 목록, 상세, 통계 화면의 조회 범위가 서로 다르게 보이지 않도록 관련 화면을 함께 점검했습니다.

## 프론트엔드 관점의 핵심

- 권한별로 보이는 데이터 범위가 달라지는 화면에서 조회 조건과 UI 상태를 함께 맞췄습니다.
- 자산 상태가 바뀌는 업무 단계에 따라 사용자가 확인해야 하는 정보와 액션을 정리했습니다.
- 외부 API 결과를 업무 화면의 성공, 실패, 예외 상태로 연결했습니다.

## 검증 관점

- 같은 검색 조건을 권한별 계정으로 조회해 목록 범위와 통계 수치가 일관적인지 확인했습니다.
- 자산 상태 변경 후 목록, 상세, 이력 화면에 같은 상태가 반영되는지 확인했습니다.
- 외부 API 실패 시 업무 데이터가 중복 저장되거나 화면이 완료 상태로 보이지 않도록 점검했습니다.

## 사용 기술

Java, Spring MVC, Spring Security, JSP, MyBatis, MariaDB, Tabulator, REST API
