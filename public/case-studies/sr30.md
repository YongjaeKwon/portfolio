# SR30 물류관리시스템

> 공개용 정리입니다. 내부 코드, 운영 URL, 환경 정보는 제외했습니다.

## 개요

Nexacro와 Spring MVC 기반 물류 운영 시스템입니다. 기존 운영 화면을 유지보수하면서 설문조사와 통계 조회 기능을 신규로 개발했습니다.

## 담당 역할

- 설문 대상자 관리 화면 구현
- 설문 응답 데이터 저장 및 조회 흐름 구현
- 통계 조회 화면과 데이터 산출 SQL 작성
- PL/SQL 작성 및 Oracle 기반 조회 쿼리 개선
- 운영 중 발견된 화면 오류와 조회 조건 이슈 수정

## 담당 기능 범위

- 설문 대상자 관리: 대상자 목록 조회, 조건 검색, 참여 상태 확인
- 설문 응답: 문항별 응답 저장, 응답 완료 여부 확인
- 통계 조회: 응답 결과 집계, 조건별 통계 조회, 운영자 확인 화면 구성
- 운영 개선: 조회 조건 오류, 화면 표시 오류, SQL 성능과 결과 정합성 점검

## 상태·예외 처리

- 설문 참여 전, 진행 중, 완료 상태를 구분해 운영자가 응답 현황을 확인할 수 있도록 구성했습니다.
- 중복 응답, 미완료 응답, 조회 조건 누락처럼 결과 통계에 영향을 줄 수 있는 케이스를 확인했습니다.
- 통계 화면은 화면 필터와 SQL 조건이 같은 기준으로 적용되는지 반복 검증했습니다.

## 프론트엔드 관점의 핵심

- 설문 참여자가 자연스럽게 응답을 완료할 수 있도록 화면 단계를 정리했습니다.
- 운영자가 대상자와 응답 결과를 확인할 수 있도록 조회 조건과 통계 화면을 구성했습니다.
- 화면 요구사항과 DB 조회 결과가 맞는지 함께 검증했습니다.

## 검증 관점

- 대상자 조건별 조회 결과와 통계 집계 결과가 같은 기준으로 계산되는지 확인했습니다.
- 설문 응답 저장 후 참여 상태와 통계 화면이 함께 갱신되는지 점검했습니다.
- 운영자가 자주 쓰는 조회 조건을 기준으로 결과 누락이나 중복이 없는지 확인했습니다.

## 사용 기술

Nexacro, Java, Spring MVC, PL/SQL, Oracle
