# TSMS / IDCMS AS 업무 시스템

> 공개용 정리입니다. 내부 코드, 고객사별 상세 정책, 운영 URL, 계정, 환경 정보는 제외했습니다.

## 프로젝트 개요

AS 접수, 상담, 이관, 배정, 배송, 개인정보 동의, QR 확인, 태블릿 전자서명 업무를 처리하는 WebSquare/JSP 기반 운영 시스템입니다. 데스크톱 관리자 화면, 모바일 접수 화면, 태블릿 서명 화면이 같은 업무 데이터를 기준으로 이어져야 했습니다.

이 프로젝트에서는 현장 사용자가 접수와 동의 단계를 진행하고, 내부 담당자가 접수 상태와 후속 처리를 확인하며, 필요 시 알림톡/BizTalk, 메일, 일정, 배송 상태까지 함께 확인하는 흐름을 다뤘습니다.

## 맡은 역할

- WebSquare/JSP 기반 데스크톱·모바일·태블릿 화면 개발 및 운영 오류 수정
- AS 접수, 이관, 배정, 상담, 진행상태 조회 화면 개선
- 개인정보 동의, QR 확인, 무인보관함 접수, 태블릿 전자서명 화면 구현
- 배송 일정, 학생 일괄 처리, 라벨/엑셀, 파일 조회·업로드 화면 개선
- Java/Spring MVC/MyBatis 기반 API와 조회 조건 점검

## 해결해야 했던 문제

AS 업무는 접수, 상담, 배정, 배송, 동의, 서명처럼 여러 단계가 이어집니다. 한 단계에서 입력된 정보가 다음 화면에서 다른 기준으로 조회되면 담당자가 처리 상태를 잘못 판단할 수 있습니다.

또한 모바일과 태블릿에서는 화면 폭과 입력 방식이 데스크톱과 달라 필수값, 파일, QR, 서명 같은 기능의 실패 케이스가 더 자주 발생할 수 있었습니다. 사용자가 어디서 막혔는지 알 수 있도록 상태와 예외를 화면에 분명히 반영하는 것이 중요했습니다.

## 구현한 기능

### AS 접수 및 이관/배정

- AS 접수 목록과 상세 화면 개선
- 접수 이관, 담당자 배정, 센터 검색 조건 처리
- 상담 및 AS 접수 흐름에서 상태별 화면 분기 개선
- 진행상태 조회 조건과 상세 화면 데이터 연결 점검

### 모바일 접수와 개인정보 동의

- 모바일 AS 접수 화면 구현 및 운영 오류 수정
- 개인정보 동의 화면, QR 기반 확인 화면, 동의 상태별 다음 단계 진입 조건 처리
- 무인보관함 모바일 접수 흐름과 접수 결과 확인 화면 개선
- 접수 번호와 사용자 정보 기준으로 후속 화면이 이어지도록 조회 조건 정리

### 태블릿 전자서명

- 태블릿 환경에서 사용하는 서명 입력 화면 개발
- 서명 저장 결과, 재시도, 완료 상태 표시 흐름 처리
- 모바일/태블릿 화면에서 동일 업무 번호 기준으로 접수, 동의, 서명 상태가 이어지는지 점검

### 일정, 배송, 알림

- 배송 일정 등록/조회 화면 개선
- 학생 일괄 처리, QR 라벨, 엑셀 다운로드 및 업로드 흐름 개선
- BizTalk/알림톡 발송 조건, 발송 결과, 예외 상태 화면 반영
- 메일/문자 등 외부 발송 결과가 사용자 화면에 남도록 상태 처리

## 기술적으로 신경 쓴 부분

현장 업무 화면은 사용자가 중간에 멈추거나 잘못 입력했을 때 복구할 수 있어야 합니다. 필수값 누락, 동의 미완료, 첨부 실패, 서명 저장 실패처럼 사용자가 직접 수정할 수 있는 오류와 시스템 연동 실패를 구분해 화면 상태를 정리했습니다.

모바일과 태블릿 화면은 같은 업무 번호와 접수 상태를 기준으로 연결되어야 했습니다. 따라서 화면 초기화, 조회 파라미터, 상세 데이터, 저장 이후 재조회 흐름이 같은 기준으로 동작하는지 반복해서 확인했습니다.

알림톡/BizTalk과 같은 외부 발송 기능은 요청 성공과 실제 발송 결과가 다를 수 있습니다. 발송 요청 이후 운영자가 확인해야 하는 상태를 화면에 반영하고, 재처리나 실패 확인이 필요한 케이스를 분리했습니다.

## 검증 방식

- 접수 유형별 필수값과 후속 단계 진입 조건 확인
- 개인정보 동의, QR 확인, 전자서명 저장 이후 진행상태가 올바르게 갱신되는지 확인
- 모바일, 태블릿, 데스크톱 화면이 같은 업무 번호를 기준으로 조회되는지 점검
- 알림톡/BizTalk 발송 성공, 실패, 대기 상태가 화면에 맞게 표시되는지 확인
- 권한 또는 센터 조건 변경 후 목록과 상세 화면의 조회 범위가 일치하는지 확인

## 배운 점

현장 업무 시스템은 화면 하나의 완성도보다 단계 간 연결이 더 중요했습니다. 접수 이후 동의, 첨부, 서명, 알림, 상태 조회가 자연스럽게 이어져야 사용자가 업무를 끝낼 수 있기 때문입니다.

이 프로젝트를 통해 모바일/태블릿 업무 화면에서 사용자의 입력 흐름과 서버 데이터 흐름을 함께 보며 개발하는 경험을 쌓았습니다. 특히 외부 발송, QR, 전자서명처럼 실패 케이스가 생기기 쉬운 기능에서 화면 상태를 세밀하게 나누는 연습이 되었습니다.

## 사용 기술

JavaScript, WebSquare, JSP, jQuery, Java, Spring MVC, MyBatis, MariaDB, REST API, BizTalk
