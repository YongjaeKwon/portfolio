# 최신 프론트엔드 보강 프로젝트 범위

> 목적: 실무에서 해온 관리자/운영 화면 경험을 React 또는 Vue 3 + TypeScript 기반으로 다시 증명하기 위한 개인 프로젝트 범위입니다. 디자인보다 기능 완성도, 상태 처리, 타입 안정성, 테스트 가능성을 우선합니다.

## 프로젝트 후보명

운영 관리자용 업무 대시보드

## 이 프로젝트가 필요한 이유

현재 실무 경험은 강하지만 WebSquare, JSP, Nexacro, Spring MVC처럼 레거시 또는 업무 시스템 기술 비중이 큽니다. 최신 프론트엔드 채용에서는 TypeScript, 상태 관리, API 캐싱, 테스트, 로딩/에러/빈 상태 처리 경험을 별도로 보여줄 필요가 있습니다.

이 프로젝트는 실무에서 해온 "등록, 업로드, 발송, 조회, 이력 확인" 흐름을 최신 프론트엔드 구조로 재구성해 보여주는 용도입니다.

## 핵심 컨셉

교육/AS/자산관리 업무를 추상화한 운영 관리자 화면입니다.

사용자는 다음 업무를 처리할 수 있어야 합니다.

- 처리 대상 목록 조회
- 기간, 상태, 담당자, 센터 기준 검색
- 상세 정보 확인
- 처리 상태 변경
- CSV 또는 엑셀 다운로드
- 메일 발송 모의 처리
- 처리 이력 타임라인 확인
- 권한별 메뉴/버튼 노출 분기

## 추천 기술 스택

1순위:

- React
- TypeScript
- Vite
- TanStack Query
- Zustand
- React Hook Form
- Zod
- MSW
- Vitest
- Playwright

대안:

- Vue 3
- TypeScript
- Pinia
- Vue Query
- VeeValidate 또는 Zod
- MSW
- Vitest
- Playwright

현재 포트폴리오 보완 관점에서는 React + TypeScript가 더 좋습니다. 실무 Vue 경험은 PPS로 이미 설명 가능하고, React/TypeScript는 최신 FE 역량 보강용으로 쓰기 좋기 때문입니다.

## 필수 화면

### 1. 대시보드

- 상태별 처리 건수 카드
- 최근 처리 이력 목록
- 기간 필터
- 담당자/센터 필터
- 실패 또는 지연 상태 하이라이트

### 2. 업무 목록

- 검색 조건: 기간, 상태, 센터, 담당자, 키워드
- 페이지네이션
- 정렬
- 다중 선택
- CSV 다운로드
- 빈 결과, 로딩, 에러 상태

### 3. 업무 상세

- 기본 정보
- 상태 변경
- 담당자 변경
- 메일 발송 모의 버튼
- 첨부파일 목록
- 처리 이력 타임라인
- 권한별 액션 버튼 분기

### 4. 등록/수정 폼

- 필수값 검증
- 날짜/상태/담당자 선택
- 저장 성공/실패 상태
- 서버 검증 오류 표시
- 변경 취소 확인

### 5. 운영 이력

- 사용자, 액션, 상태, 시간 기준 검색
- 상세 이력 확인
- 실패 이력 재처리 모의 버튼

## 반드시 구현할 상태

- loading
- empty
- error
- success
- optimistic update 또는 refetch
- validation error
- permission denied
- stale data
- retryable failure

## 테스트 범위

### Vitest

- 검색 조건을 query parameter로 변환하는 함수
- 상태값에 따른 액션 가능 여부 판단 함수
- 폼 검증 스키마
- 다운로드 파일명 생성 함수

### Playwright

- 목록 조회 후 필터 변경
- 상세 진입 후 상태 변경
- 등록 폼 필수값 검증
- 권한별 버튼 노출 여부
- 에러 상태 재시도

## 포트폴리오에 넣을 문장

> 실무에서 경험한 관리자 화면의 등록, 조회, 상태 변경, 이력 확인 흐름을 React/TypeScript 기반으로 재구성한 프로젝트입니다. TanStack Query로 서버 상태를 관리하고, MSW로 API를 모의 처리하며, Vitest와 Playwright로 검색 조건, 폼 검증, 권한별 액션 노출을 검증했습니다.
